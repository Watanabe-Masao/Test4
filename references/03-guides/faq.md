# FAQ（よくある質問）

---

## インポート関連

### Q: 対応しているファイル形式は？

Excel (.xlsx) と CSV (.csv) に対応しています。Shift_JIS / UTF-8 の自動判定に対応。

| ファイル種別 | 形式 | 説明 |
|---|---|---|
| 分類別売上 | CSV | 日別・分類別の売上データ（基準値） |
| 仕入 | Excel (.xlsx) | 日別・仕入先別の原価・売価 |
| 花 | Excel (.xlsx) | 花部門の売上（原価・売価・客数） |
| 産直 | Excel (.xlsx) | 産直部門の売上（原価・売価） |
| 移動 | Excel (.xlsx) | 店舗間移動・部門間移動 |
| 時間帯 | CSV | 分類別時間帯売上（カテゴリ x 時間帯） |
| 部門 KPI | Excel/CSV | 部門別の予算・実績・粗利率等 |

### Q: ファイルをドロップしても認識されない

ファイル名に種別を示すキーワード（仕入、売上、値引 等）が含まれているか確認してください。
詳細は `references/03-guides/operations.md` および `references/03-guides/operations.md` のトラブルシューティングを参照。

### Q: 複数ファイルを同時にインポートできる？

はい。ドロップゾーンに複数ファイルを同時にドラッグ & ドロップできます。
ファイル種別は自動判定されます。

### Q: データを更新したい場合は？

同じ種別のファイルを再度インポートしてください。
差分確認モーダルが表示され、上書きするか維持するかを選択できます。

### Q: 前年データの比較はどうやる？

1. 前年の売上・売変ファイルをインポート
2. 管理画面の「前年マッピング設定」で店舗の対応付けを確認
3. ダッシュボードの前年比較ウィジェットが自動表示

分類別時間帯売上の前年分は `9.分類別時間帯売上.csv` としてインポートしてください。

## 計算結果の解釈

### Q: 在庫法と推定法の違いは？

| 項目 | 在庫法（invMethod） | 推定法（estMethod） |
|---|---|---|
| スコープ | 全売上・全仕入 | 在庫販売のみ（花・産直除外） |
| 計算式 | 売上原価 = 期首在庫 + 総仕入 - 期末在庫 | 推定原価 = 粗売上 x (1 - 値入率) + 消耗品費 |
| 目的 | 実績粗利の確認 | 推定在庫と実績の比較（異常検知） |

在庫法の計算式:
```
粗利益 = 総売上高 - 売上原価
売上原価 = 期首在庫原価 + 総仕入原価 - 期末在庫原価
粗利率 = 粗利益 / 総売上高
```

推定法の計算式:
```
推定原価 = 粗売上 x (1 - 値入率) + 消耗品費
マージン = コア売上 - 推定原価
マージン率 = マージン / コア売上
```

両方の結果を比較することで、計算の妥当性を相互検証できます。

詳細は `references/03-guides/calculation-engine.md` を参照。

### Q: 粗利率の計算式は？

```
粗利益 = 総売上高 - 売上原価
粗利率 = 粗利益 / 総売上高
```

各指標の計算式は、KPI カードをクリックすると表示される説明パネルで確認できます。

### Q: 要因分解とは？

売上変動を「客数効果」「点数効果」「単価効果」「構成比効果」に分解する分析手法です。
シャープリー値に基づき、各要因の寄与度を公平に配分します。

**数学的保証**: 各効果の合計は必ず実際の売上差（当年売上 - 前年売上）に一致します。

分解レベル:
- **2 要素分解**: 客数効果 + 客単価効果 = 売上差
- **3 要素分解**: 客数効果 + 点数効果 + 単価効果 = 売上差
- **5 要素分解**: 客数効果 + 点数効果 + 価格効果 + 構成比効果 = 売上差

### Q: 値入率とは？

値入率（マークアップ率）は、仕入原価に対する利益の割合です。

```
値入率 = (売価 - 原価) / 売価
```

- **平均値入率**: 全品目の加重平均
- **コア値入率**: 花・産直を除いた値入率

### Q: 売変率とは？

売変率（ディスカウント率）は、売上に対する値引き額の割合です。

```
売変率 = |売変額| / 粗売上
```

売変は 4 種類（71-74 コード）に分類されます。

## データソースの乖離

### Q: 「売上データの乖離が 1% を超えています」という警告が出ました。問題ありますか？

これは仕様上の想定内の動作です。

本システムの売上データは複数のファイルに由来します:

- **classifiedSales**（分類別売上 CSV）: 総売上・売変・客数の基準値
- **categoryTimeSales**（分類別時間帯 CSV）: 時間帯分析・カテゴリドリルダウン用

これらは以下の理由により数%乖離することがあります:

1. **丸め誤差**: 集計時の端数処理の違い
2. **集計タイミング差**: データ抽出時刻の違い
3. **対象範囲差**: 集計対象の微妙な定義の違い

計算パイプラインは常に `classifiedSales` 由来の値（`StoreResult.totalSales`）を
基準値として使用するため、計算結果の整合性は保たれています。

時間帯分析チャートの数値が KPI カードの売上合計と若干異なる場合がありますが、
これはデータソースの違いによるものであり、バグではありません。

## DuckDB チャート関連

### Q: DuckDB チャートが表示されない

DuckDB-WASM はデータロード後数秒で初期化されます。以下を確認してください:

1. 分類別時間帯売上データがインポート済みか
2. ブラウザが WebAssembly に対応しているか（Chrome 91+, Firefox 89+, Safari 15+）
3. ページを再読み込みしてエンジンの再初期化を試す
4. ブラウザの開発者ツール（F12）でコンソールのエラーメッセージを確認

DuckDB が利用不可の場合は従来の JavaScript パスにフォールバックします。

### Q: DuckDB チャートと既存チャートで数値が異なります。なぜですか？

DuckDB チャートは SQL クエリで直接データを集約するため、
既存の JavaScript 計算パイプラインとは異なるコードパスを通ります。

- **小さな差異（< 0.1%）**: 浮動小数点の丸め順序の違いによるもの。問題なし
- **大きな差異**: バグの可能性があります。開発チームに報告してください

### Q: 「X-Y のロードに失敗（スキップ）」というメッセージが出ます

IndexedDB に保存された過去月のデータ読み込みに失敗した場合に表示されます。
該当月のデータを再インポートすることで解決します。当月のデータには影響しません。

## パフォーマンス

### Q: データが多いとアプリケーションが重くなります。対処法は？

以下の対策を試してください:

1. **店舗数を絞る**: 店舗選択フィルタで分析対象店舗を限定する
2. **期間を短くする**: DateRangePicker で分析対象期間を絞る
3. **不要なウィジェットを非表示にする**: クエリ実行回数を減らせます
4. **ブラウザのタブを整理する**: 他のタブのメモリ消費が DuckDB-WASM に影響します

### Q: 推奨される同時ロード月数は？

3-6 ヶ月程度を推奨します。それ以上の月数をロードすると、
メモリ消費量が増大しパフォーマンスが低下する可能性があります。

### Q: マテリアライズとは？

`store_day_summary` VIEW は 6 テーブルの LEFT JOIN で構成されており、
クエリのたびに結合処理が実行されます。データ量が大きい場合、
`materializeSummary()` を呼び出すことで VIEW の結果をテーブルに事前計算できます。
ただし、データ更新のたびにマテリアライズを再実行する必要があります。

## ブラウザ対応状況

### Q: どのブラウザで動作しますか？

| ブラウザ | 対応状況 | 備考 |
|---|---|---|
| **Chrome** (最新版) | 推奨 | 最もテストされている環境 |
| **Edge** (最新版) | 推奨 | Chrome と同等の動作 |
| **Firefox** (最新版) | 対応 | 大量データ時に若干のパフォーマンス差がある場合あり |
| **Safari** (最新版) | 対応 | IndexedDB の容量制限に注意 |
| **モバイルブラウザ** | 非推奨 | メモリ不足でクラッシュする可能性あり。デスクトップブラウザでの使用を推奨 |
| **Internet Explorer** | 非対応 | WebAssembly 非対応のため動作不可 |

### Q: オフラインで使える？

初回アクセス時に静的アセットがキャッシュされれば、以降はオフラインで使用可能です。
全ての計算はブラウザ内で完結します。

### Q: データはどこに保存される？サーバーに送信される？

全てのデータは**ブラウザの IndexedDB に保存**されます。
サーバーへのデータ送信は一切行いません。
詳細は `references/03-guides/security.md` を参照してください。

## データの管理

### Q: ブラウザを閉じてもデータは残る？

はい。IndexedDB に永続化されているため、同じブラウザで再度アクセスすると
復元を確認するモーダルが表示されます。

### Q: データを完全に削除したい

管理画面 -> 「ストレージ管理」タブ -> 「データを削除」ボタンで IndexedDB をクリアできます。

### Q: 異なるブラウザ間でデータを共有できる？

直接の共有機能はありません。各ブラウザの IndexedDB は独立しています。
同じファイルを別のブラウザでインポートしてください。

### Q: 印刷できる？

レポートページで `@media print` に対応した印刷用レイアウトが提供されます。
ブラウザの印刷機能（Ctrl+P / Cmd+P）から印刷してください。
