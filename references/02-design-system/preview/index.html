仕入粗利管理ツール — Design System v2.1

本体コード Watanabe-Masao/Test4 の app/src/presentation/theme/ 配下 (tokens.ts / theme.ts / semanticColors.ts / colorSystem.ts) を正本として、その語彙を外部に説明する documentation layer です。 左のナビから Foundations・Components・Patterns を順に確認できます。

v2.1 の立場

v2.0 は「本体に提案するトークン仕様」として書かれていましたが、実装を確認した結果、 本体は既に意味層 (theme.colors.* / interactive.* / chart.semantic.*) と ライト/ダーク両テーマを備えていました。v2.1 は提案書ではなく、 本体実装の外部 documentation として書き直したものです。 本体コードの変更は伴いません。詳細は docs/v2-to-v2.1-changes.md。

トークン層

Layer 1 --c-* (palette: tokens.ts) の上に、 Layer 2 --theme-* (theme.ts の darkColors/lightColors) と 業務概念色 --chart-* (chart.semantic.*)、 取引区分グラデーション --gradient-* (categoryGradients: ti/to/bi/bo 他) が乗ります。 トレンド UI には CUD-safe の --c-positive/negative/caution (semanticColors.ts の sc.*) を使用。

目次

Foundations 色・タイポグラフィ・余白・角丸・シャドウ・モーションの基礎トークン。Category Gradients は取引区分 (ti/to/bi/bo) のページに。
Components Button から Toast まで 13 コンポーネント。Layer 2 トークン参照のみで構築。
Chart Semantic 業務概念 (売上・粗利・客数・売変 …) ごとの色マップ。Test4 の中央設定を可視化。
Patterns ダッシュボード・データインポートなど、実際の画面構成例。
UI Kit React のクリックスルー試作機。ダッシュボード全体を通しで確認できます。
