# RM-004 — 値引き ReadModel

## 1. 概要

`buildDiscountFactReadModel(rows)` は **pure 同期** builder で、`DiscountFactRow[]`（typeCode 71-74 の値引き内訳）を grand 合計 + dept / store / daily の 4 view に集約する正本（`discount-definition.md` 準拠）。

## 2. Input Contract

- `rows: readonly DiscountFactRow[]` — Zod parse 済み（`DiscountFactQueryInput.rowSchema`）
- 各 row: `date` / `storeId` / `deptCode` / `typeCode` (71-74) / `discountAmount` / `discountQty`

`typeCode` 意味:
- 71: 売変（売価変更）
- 72: 廃棄
- 73: 返品
- 74: その他値引き

## 3. Output Contract

`DiscountFactReadModel`:

- `grandTotals`: `totalDiscountAmount` / `totalDiscountQty`
- `byTypeCode`: 4 typeCode 別の合計 + 内訳率
- `byStore` / `byDept` / `daily` view
- `meta.sourceDate`

派生 helper:
- `toStoreDiscountRows(model)` — store-level
- `toDailyDiscountRows(model)` — daily
- `toDeptDiscountRows(model)` — dept-level

## 4. Upstream Dependencies

- `discountFactHandler` (`QueryHandler<DiscountFactInput, DiscountFactOutput>`)
- DuckDB `discount_fact_*` SQL

## 5. Consumers

- `useWidgetDataOrchestrator` で配布
- `IntegratedSalesChart` の「売変」モード
- `WaterfallChartWidget`（粗利分解の値引き component）

## 6. Fallback / Readiness

- 入力 0 件 → 全 totals = 0
- typeCode 別 view も 0 件で安全（`byTypeCode[71] = 0` など）

## 7. Invariants

- INV-DF-01: `grandTotals.totalDiscountAmount` = sum(rows.discountAmount)
- INV-DF-02: `byTypeCode[71] + [72] + [73] + [74]` = `grandTotals.totalDiscountAmount`（typeCode 完全網羅）
- INV-DF-03: 4 view の合計値は grand と一致

詳細: `discount-definition.md` §「typeCode 71-74 の意味」
