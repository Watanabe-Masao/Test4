# RM-009 — 自由期間部門 KPI ReadModel

## 1. 概要

`buildFreePeriodDeptKPIReadModel(rawInput, options)` は **pure 同期** builder で、月別部門 KPI を期間 KPI に統合し、率系指標を `weightedAverageRate`（line 59）で集計する正本（`free-period-budget-kpi-contract.md` 準拠）。SQL 側で率計算を行わず加重和のみ transport する `noRateInFreePeriodSqlGuard` 契約に準拠。

## 2. Input Contract

- `rawInput: readonly RawDeptKPIInput[]` — 月別部門 KPI 行（年/月/店/部門/客数/客単価/粗利率 等）
- `options.dateRange`: 集約対象期間
- `options.selectedStoreIds`: store filter

派生関数:
- `weightedAverageRate(numerator, denominator)` (line 59) — 加重平均率の正本 helper（SQL から剥離）
- `dateRangeToYearMonths(dateRange)` (line 104) — `dateRange` → `[year, month]` 配列展開

## 3. Output Contract

`FreePeriodDeptKPIReadModel`:

- `byDept`: 部門別 KPI（`customerCount` / `unitPrice` / `grossProfitRate` 等）
- `byStore`: 店別 KPI
- `total`: 全部門 grand
- 率系指標は全て `weightedAverageRate` 経由で集計（単純平均ではない）
- `meta.aggregatedFrom`: 元月別 KPI の参照範囲

## 4. Upstream Dependencies

- `freePeriodDeptKPIHandler` (`@/application/queries/freePeriodDeptKPIHandler.ts`)
- 月別 KPI table（DuckDB `dept_kpi_monthly_*`）
- SQL は加重和のみ計算 → builder で `weightedAverageRate` 適用

## 5. Consumers

- `useFreePeriodAnalysis` 経由
- 自由期間部門ベンチマーク chart

## 6. Fallback / Readiness

- 0 件 → `byDept` / `byStore` 空、`total` 全 0
- `denominator = 0` の率計算 → `weightedAverageRate` は 0 を返す（除算回避）
- 単月のみ（dateRange = 1 month）→ 通常通り集計（month-agnostic）

## 7. Invariants

- INV-FPK-01: 率系指標は `weightedAverageRate(sum(numerator), sum(denominator))` で計算
- INV-FPK-02: `byDept` / `byStore` の客数合計は `total.customerCount` と一致
- INV-FPK-03: SQL 側で率を計算しない（`noRateInFreePeriodSqlGuard` 契約）
- INV-FPK-04: `dateRangeToYearMonths` の出力は dateRange の月集合と完全一致

詳細: `free-period-budget-kpi-contract.md` §「率系指標の加重平均化」

## 8. Co-Change Impact

- `RawDeptKPIInput` schema 変更 → builder 破壊
- `weightedAverageRate` の signature 変更 → 全率系計算経路の破壊
- 率系指標の追加（粗利額率など）→ INV-FPK-01 を新指標にも適用必要
- SQL 側で率計算を行う変更 → `noRateInFreePeriodSqlGuard` 違反
