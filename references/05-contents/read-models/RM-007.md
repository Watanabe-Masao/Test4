# RM-007 — 自由期間分析 ReadModel

## 1. 概要

`buildFreePeriodReadModel(rows, options)` は **pure 同期** builder で、自由期間（任意 dateRange）の売上・客数・粗利を `FreePeriodReadModel` に集約する正本（`free-period-analysis-definition.md` 準拠）。月境界に依存しない arbitrary period の分析を可能にする。

## 2. Input Contract

- `rows`: `freePeriodHandler` 経由で取得した raw row 配列（Zod parse 済み）
- `options.budget`（optional）: 期間中の予算情報（`prorateBudget` で日割り適用）
- `options.previousSummary`（optional）: 前年同期比較用 summary

派生関数（同 module）:
- `computeFreePeriodSummary(rows)` (line 21) — summary のみ計算
- `prorateBudget(budget, days)` (line 90) — 月予算 → 期間予算に按分

## 3. Output Contract

`FreePeriodReadModel`:

- `summary`: `totalSales` / `totalCustomers` / `totalGrossProfit` 等の grand
- `daily`: 日別 row 配列
- `byStore` / `byDept`: store / dept 別集計
- `budget`（optional）: 按分済み期間予算
- `previous`（optional）: 前年比較 summary
- `meta`: `dateRange` / `dayCount` / `selectedStoreIds`

## 4. Upstream Dependencies

- `freePeriodHandler` (`QueryHandler<FreePeriodQueryInput, FreePeriodReadModel>`、`@/application/queries/freePeriodHandler.ts`)
- DuckDB `free_period_*` SQL（month-agnostic な dateRange filter）

## 5. Consumers

- `useFreePeriodAnalysis` hook 経由
- `FreePeriodAnalysisChart` 等の自由期間 chart
- `selectPrevYearSummaryFromFreePeriod` adapter（PR Phase 6 Step A、`features/comparison`）

## 6. Fallback / Readiness

- query 自体が reject → handler 側で `status: 'error'`
- 0 件結果 → `summary` 全 0、`daily` 空配列
- budget 不在時 → `budget` field omitted（optional）
- previousSummary 不在時 → `previous` field omitted（optional）

## 7. Invariants

- INV-FP-01: `summary.totalSales` = sum(`daily[].sales`)
- INV-FP-02: `summary.totalCustomers` = sum(`daily[].customers`)
- INV-FP-03: `byStore` / `byDept` 合計は `summary` と一致
- INV-FP-04: 期間 day count = `dateRange.dayCount`（zero-day period 不可）

詳細: `free-period-analysis-definition.md` §「自由期間 vs 単月の意味分離」

## 8. Co-Change Impact

- `FreePeriodReadModel` 構造変更 → 全 consumer 破壊
- `freePeriodHandler` の SQL 変更 → row schema の Zod parse 修正
- `prorateBudget` の按分ロジック変更 → 期間予算計算の意味変化（INV-FP-04 不変条件確認必要）
- `selectPrevYearSummaryFromFreePeriod` adapter の signature 変更 → 比較計算の経路破壊
