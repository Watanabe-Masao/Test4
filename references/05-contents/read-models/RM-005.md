# RM-005 — 客数 ReadModel

## 1. 概要

`buildCustomerFactReadModel(rows)` は **pure 同期** builder で、`CustomerFactDailyRow[]` を `CustomerFactReadModel` (`grandTotalCustomers` + dept / store / daily / store-day view) に集約する正本（`customer-definition.md` 準拠）。`StoreResult.totalCustomers` の **正本上書き先**。

## 2. Input Contract

- `rows: readonly CustomerFactDailyRow[]` — Zod parse 済み
- 各 row: `date` / `storeId` / `deptCode` / `customerCount`

## 3. Output Contract

`CustomerFactReadModel`:

- `grandTotalCustomers`: 全期間 / 全店 / 全部門の合計（`StoreResult.totalCustomers` の上書き先）
- `byStore` / `byDept` / `daily` / `byStoreDay` の 4 view
- `meta.sourceDate`

派生 helper:
- `toStoreCustomerRows(model)` — `ReadonlyMap<storeId, customerCount>`
- `toDailyCustomerRows(model)` — `ReadonlyMap<dateKey, customerCount>`
- `toStoreDayCustomerRows(model)` — store × day matrix

`selectors.ts` (同 module、SP-B ADR-B-003 で抽出) の selector 群:
- `selectTotalCustomers(slice, fallback)` — ready なら `grandTotalCustomers`、未 ready なら fallback
- `selectStoreCustomerMap(slice)` — ready なら `toStoreCustomerRows`
- `selectCustomerCountOrUndefined(slice)` — ready なら `grandTotalCustomers`、未 ready なら undefined

## 4. Upstream Dependencies

- `customerFactHandler` (`QueryHandler<CustomerFactInput, CustomerFactOutput>`、line 73 同 file 内 export)
- DuckDB `customer_fact_*` SQL

## 5. Consumers

- `useWidgetDataOrchestrator` 経由 `ctx.readModels.customerFact` として配布
- `PerformanceIndexChart`（PI 値・偏差値の母数）
- `SensitivityDashboard`（客数感度分析）
- `CategoryPerformanceChart`（カテゴリ別 PI）
- `analysis-causal-chain` widget（因果チェーン）

## 6. Fallback / Readiness

- discriminated union: `{ status: 'pending' | 'ready' | 'error', data?: CustomerFactReadModel }`
- consumer は `selectors.ts` 経由で `status === 'ready'` 判定 + fallback 値選択
- legacy fallback 経路: `result.totalCustomers`（StoreResult から）— `selectTotalCustomers` で吸収

## 7. Invariants

- INV-CF-01: `grandTotalCustomers` = sum(rows.customerCount)
- INV-CF-02: `byStore` / `byDept` / `daily` の各合計は `grandTotalCustomers` と一致
- INV-CF-03: ready 時、`grandTotalCustomers` は `result.totalCustomers` の正本上書き先（PI 値計算で母数として使用）

詳細: `customer-definition.md` §「客数 vs 販売点数の意味分離」
