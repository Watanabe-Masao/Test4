# RM-008 — 自由期間予算 ReadModel

## 1. 概要

`buildFreePeriodBudgetReadModel(rawBudgets, options)` は **pure 同期** builder で、月別予算データを期間予算に按分（`prorateBudgetForPeriod`、line 35）した上で `FreePeriodBudgetReadModel` に統合する正本（`free-period-budget-kpi-contract.md` 準拠）。

## 2. Input Contract

- `rawBudgets: readonly RawBudgetInput[]` — 月単位の budget 行（年/月/店/部門/予算金額）
- `options.dateRange`: 按分対象期間（`dateFrom` / `dateTo`）
- `options.selectedStoreIds`: store filter（不在なら全店）

派生関数:
- `prorateBudgetForPeriod(monthlyBudget, period)` — 月予算 × (期間日数 / 月日数) で按分

## 3. Output Contract

`FreePeriodBudgetReadModel`:

- `totalBudget`: 期間予算合計（按分済み）
- `byStore` / `byDept`: store / dept 別予算
- `daily`: 日割り予算（均等按分）
- `meta.proratedFrom`: 元月別予算の参照範囲（按分の透明性）

## 4. Upstream Dependencies

- `freePeriodBudgetHandler` (`@/application/queries/freePeriodBudgetHandler.ts`)
- `RawBudgetInput` 月別正本（KPI table）

## 5. Consumers

- `useFreePeriodAnalysis` 経由（`FreePeriodReadModel.budget` field を populate する path）
- 予算達成率を表示する自由期間 chart

## 6. Fallback / Readiness

- 月別予算 0 件 → `totalBudget = 0`、`daily` 空配列（pure builder のため fail せず）
- store filter 適用後 0 件 → 同上

## 7. Invariants

- INV-FPB-01: `totalBudget` = sum(prorate された月別予算)
- INV-FPB-02: `byStore` / `byDept` 合計は `totalBudget` と一致
- INV-FPB-03: `daily` の合計 = `totalBudget`（按分の保存則）
- INV-FPB-04: 期間日数 = 0 → `totalBudget = 0` の安全動作

詳細: `free-period-budget-kpi-contract.md` §「按分の意味」
