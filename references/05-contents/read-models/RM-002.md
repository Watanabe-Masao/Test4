# RM-002 — 粗利 ReadModel

## 1. 概要

`calculateGrossProfit(input)` は **pure 同期** 関数で、入力 `GrossProfitInput`（売上・仕入・値引き）から **4 種の粗利**（在庫法×2 / 推定法×2）と粗利率を導出する正本（`gross-profit-definition.md` の 4 種定義に対応）。

## 2. Input Contract

- `GrossProfitInput`: `totalSales` / `totalCost` / `totalDiscount` / `costInclusionAmount` / `inventoryAdjust`（optional）
- 入力は呼び出し側で fact 統合済み（本関数は計算のみ、I/O なし）
- `fallback` variant `calculateGrossProfitWithFallback` は `purchaseCost` 不在時の estimate path を持つ（line 105〜）

## 3. Output Contract

`GrossProfitReadModel` は以下を保証:

- `inventoryMethod`: 在庫法粗利（`totalSales - totalCost - totalDiscount + costInclusionAmount`）と粗利率
- `estimationMethod`: 推定法粗利（在庫調整を含めた式）と粗利率
- `inventoryMethodWithAdjust` / `estimationMethodWithAdjust`: 在庫調整適用後の 2 種
- 4 種いずれも `value` + `rate` のペア
- `grossProfitFromStoreResult(result)` adapter (line 161) は legacy `StoreResult` 互換を提供

## 4. Upstream Dependencies

- `domain/calculations/grossProfit/*` の WASM/TS bridge（`calculateGrossProfit` 内部で WASM 経由）
- WASM 不在時は TypeScript fallback（A2 純粋計算層）

## 5. Consumers

- `useWidgetDataOrchestrator` 経由で全 widget に配布
- `useUnifiedWidgetContext` の `result.grossProfit*` field 構築
- `WaterfallChartWidget` / `GrossProfitHeatmapWidget` / `RegressionInsightChart` 等の chart

## 6. Fallback / Readiness

- WASM bridge が ready なら WASM、そうでなければ TS fallback（domain 純粋関数のミラー）
- 入力が部分欠損（`costInclusionAmount` 不在等）は `calculateGrossProfitWithFallback` に分岐

## 7. Invariants

- INV-GPD-01: 4 種粗利率は `value / totalSales` で算出（`totalSales = 0` のとき `rate = 0`）
- INV-GPD-02: WASM 結果と TS fallback の数値整合（`shapleyIdentityInvariant.test.ts` 系で観測）

詳細: `gross-profit-definition.md` §「4 種の意味と式」
