# RM-010 — 月次前年売上 ReadModel

## 1. 概要

`selectMonthlyPrevYearSales(input, mode)` は **pure 同期** selector で、月次 KPI から前年売上を `sameDate` / `sameDow` の 2 mode で射影する正本。`MonthlyPrevYearSalesProjection` schema (Zod) を返す。

## 2. Input Contract

- `input`: 月次 KPI 行（`year` / `month` / `prevYearSalesSameDate` / `prevYearSalesSameDow`）
- `mode: MonthlyPrevYearSalesMode` — `'sameDate'` | `'sameDow'`（同日比較 / 同曜日比較）

`mode` 意味:
- `sameDate`: 単純な前年同月同日（カレンダー比較）
- `sameDow`: 前年同曜日（曜日合わせ、wow 補正）

## 3. Output Contract

`MonthlyPrevYearSalesProjection` (Zod schema):

- `value`: 選択された前年売上値（mode に応じた射影）
- `source: MonthlyPrevYearSalesSource` — `'kpi-sameDate'` | `'kpi-sameDow'` | `'none'`（射影元の透明性）
- 入力 KPI に当該 mode の値が無い → `source: 'none'`、`value: null`

## 4. Upstream Dependencies

- 月次 KPI table（直接的な query handler は持たず、上位 hook が KPI から本 selector に投入）
- `MonthlyPrevYearSalesModeSchema` / `MonthlyPrevYearSalesSourceSchema` Zod enum

## 5. Consumers

- `selectPrevYearSummaryFromFreePeriod` の補完
- 月次比較 chart で「前年同期」値の取得経路として利用
- ConditionSummary 系 widget が前年比較値を必要とするときに参照

## 6. Fallback / Readiness

- 月次 KPI が `sameDate` のみで `sameDow` 不在 → mode=`sameDow` なら `source: 'none'`
- 両 mode 不在 → `value: null`、`source: 'none'`（明示的 absence、null safety）

## 7. Invariants

- INV-MPY-01: `mode` ∈ enum、enum 外は Zod parse で reject
- INV-MPY-02: `source: 'none'` ⟺ `value: null`（透明性の保存）
- INV-MPY-03: `mode === 'sameDate'` で KPI に値があれば `source: 'kpi-sameDate'`、`mode === 'sameDow'` で値があれば `source: 'kpi-sameDow'`
