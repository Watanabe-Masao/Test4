# RM-006 — 要因分解 ReadModel

## 1. 概要

`calculateFactorDecomposition(input)` は **pure 同期** 関数で、Shapley 5 要素分解（販売数量×客単価×粗利率×値引率×構成比）の正本（`authoritative-calculation-definition.md` + `invariant-catalog.md` の Shapley 不変条件 INV-SHAPLEY-* に準拠）。

## 2. Input Contract

- `DecomposeInput`: 当期 / 比較期の `salesAmount` / `salesQty` / `unitPrice` / `grossProfit` / `discountAmount` / `compositionRatio`
- 比較期データ欠損時は `option.fallback` を許容（前年データなし → `decompose5` ではなく単純差分にフォールバック）

## 3. Output Contract

`FactorDecompositionResult`:

- `total`: 当期 - 比較期 の差分（売上 / 客数 / 粗利 等）
- `factors`: `quantity` / `unitPrice` / `grossProfitRate` / `discountRate` / `compositionRatio` の 5 要素貢献度
- 不変条件: `sum(factors) === total`（Shapley 恒等式、`shapleyIdentityInvariant.test.ts` で検証）
- WASM 実装が ready なら `factor-decomposition-wasm` 経由、未 ready なら TS fallback

## 4. Upstream Dependencies

- `domain/calculations/factorDecomposition/*` の WASM bridge
- WASM 不在時は TS pure mirror

## 5. Consumers

- `useWidgetDataOrchestrator` 経由
- `WaterfallChartWidget` (粗利分解 chart)
- `DecompositionTabContent` (Insight ページ)
- `analysis-causal-chain` widget (因果チェーン分析)
- `RegressionInsightChart` (回帰分析の入力)

## 6. Fallback / Readiness

- 当期 / 比較期の片方欠損 → `option.fallback === 'simple-diff'` で単純差分計算
- WASM bridge ready check、未 ready なら TS pure に自動 fallback
- 入力 0 → `{ total: 0, factors: { ...全 0 } }`

## 7. Invariants

- INV-SHAPLEY-2: 2 要素 Shapley 恒等式
- INV-SHAPLEY-3: 3 要素 Shapley 恒等式
- INV-SHAPLEY-5: 5 要素 Shapley 恒等式（本 readModel の主不変条件）
- INV-FACTOR-NONNEG: `compositionRatio` は [0, 1] の範囲

詳細: `invariant-catalog.md` §Shapley、`authoritative-calculation-definition.md`
