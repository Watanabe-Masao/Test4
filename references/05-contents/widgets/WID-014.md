# WID-014 — 着地予測・ゴールシーク

## 1. 概要

`ForecastToolsWidget` に full ctx を passthrough、storeKey を React key として指定する registry entry（registryExecWidgets.tsx:68-74）。

## 2. Data Acquisition Path

- registry 行では `<ForecastToolsWidget key={ctx.storeKey} ctx={ctx} />` の形
- 実際のデータ取得・予測計算は子 `ForecastToolsWidget` 内部に委譲

## 3. Context Fields Consumed — 解説

- `storeKey` は React key 用にのみ直接参照（store 切替で再マウント）
- 実ctx 依存は子側に散在

## 4. Composition

- 直接の子: `ForecastToolsWidget`（`./ForecastTools` から import、:2）
- `ForecastTools.tsx` が単独 file として存在

## 5. Visibility

- `isVisible` 未指定（常時可視）

## 6. Invariants

- 本 widget 自身は数学的不変条件を持たない
- 予測計算の不変は子 `ForecastToolsWidget` の `calculateForecast` 等が担当（inquiry/05 関連）

## 7. Co-Change Impact

- `storeKey: string` 型変更
- `ForecastToolsWidget` の props shape 変更
- `linkTo` の view / tab 型変更

## 8. Guard / Rule References

- `responsibilityTagGuard`（R:component）
- `presentationIsolationGuard`
- `pageMetaGuard`

## 9. Pipeline Concerns / Upstream Requests
