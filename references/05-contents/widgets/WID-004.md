# WID-004 — 時間帯×曜日ヒートマップ

## 1. 概要

`UnifiedHeatmapWidget` に full ctx を passthrough し、時間帯 × 曜日のヒートマップを描画する registry entry（registryChartWidgets.tsx:92-98）。

## 2. Data Acquisition Path

- registry 行では `ctx` を full passthrough（`acquisitionPath: ctx-direct`）
- 実際のデータ取得・描画は全て `UnifiedHeatmapWidget` 内部に委譲
- `isVisible` に **関数参照** `isTimeSeriesVisible`（`widgetVisibility.ts` から import）を指定

## 3. Context Fields Consumed — 解説

- registry 行からの直接参照 0（full ctx passthrough）
- 実 ctx 依存は子 `UnifiedHeatmapWidget` 側に散在（inquiry/02 §D-1 ambiguity）
- `isVisible` predicate は外部関数のため predicate 本体の判定条件は本 spec で未追跡

## 4. Composition

- 直接の子: `UnifiedHeatmapWidget`（`./UnifiedAnalyticsWidgets` から import、:8）
- `UnifiedAnalyticsWidgets.tsx` 内に `UnifiedHeatmapWidget` と `UnifiedStoreHourlyWidget` が同居

## 5. Visibility

- 条件: `isVisible: isTimeSeriesVisible`（`widgetVisibility.ts` の関数参照）
- predicate 本体は本 spec では未追跡。`widgetVisibility.ts` を直接参照のこと

## 6. Invariants

- 本 widget 自身は数学的不変条件を持たない
- 時間帯データの集計不変は子 `UnifiedHeatmapWidget` 経由で `timeSlotLane` / `projectTimeSlotSeries` に従属（CLAUDE.md directly 主要変更 Phase 6 Step C）
