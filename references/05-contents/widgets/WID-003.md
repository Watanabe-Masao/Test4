# WID-003 — 粗利推移チャート

## 1. 概要

`GrossProfitAmountChart` に daily / budget / prevYear + `buildPrevYearCostMap` 近似値を forward し、粗利額の時系列を描画する registry entry（registryChartWidgets.tsx:62-82）。

## 2. Data Acquisition Path

- `ctx.result.daily` / `ctx.result.grossProfitBudget` / `ctx.prevYear.daily` を個別 props に展開
- `chartPeriodProps?.rangeStart` / `rangeEnd` で range scope を子に渡す
- **同ファイル内 helper `buildPrevYearCostMap(ctx)`**（`registryChartWidgets.tsx:18-27`）を呼び出し、前年日別近似原価 map を合成して `prevYearCostMap` prop として注入

## 3. Context Fields Consumed — 解説

- 11 field を touch。WID-002 の 17 より少ないが `chartPeriodProps.rangeStart` / `rangeEnd` の sub-path と `prevYear.hasPrevYear`/`daily` の conditional forward を含む
- `prevYear.hasPrevYear ? prevYear.daily : undefined` の optional 転送で、子が前年不在を判定可能
- `targetRate` / `warningRate` は粗利率の警告閾値を子に渡す

## 4. Composition

- 直接の子: `GrossProfitAmountChart`（`@/presentation/components/charts` から import、`registryChartWidgets.tsx:3`）
- **inline helper**: `buildPrevYearCostMap(ctx)` が registry file 内に定義され、本 widget の render で呼ばれる（他 widget は呼ばない）

## 5. Visibility

- `isVisible` 未指定（常時可視）
- 前年データが無い場合は `prevYearCostMap` が `undefined` を返し、子が内部で handle

## 6. Invariants

- 本 widget 自身は数学的不変条件を持たない
- `buildPrevYearCostMap` は「前年が `totalSales > 0` の場合のみ map を返す」事実（registryChartWidgets.tsx:20）、「`entry.sales - entry.discount` で日別 cost 近似」（:24）
