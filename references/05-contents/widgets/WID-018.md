# WID-018 — PI値・偏差値・Zスコア

## 1. 概要

`PerformanceIndexChart` に 16 field を forward、そのうち `totalCustomers` と `storeCustomerMap` を **IIFE（即時関数）内で readModels.customerFact の ready 判定込みで導出**する registry entry（registryAnalysisWidgets.tsx:55-85）。

## 2. Data Acquisition Path

- 16 field を個別 props に展開
- **2 箇所の IIFE（即時実行関数）**:
  - `totalCustomers`: `(() => { const cf = ctx.readModels?.customerFact; return cf?.status === 'ready' ? cf.data.grandTotalCustomers : ctx.result.totalCustomers })()`
  - `storeCustomerMap`: `ctx.readModels?.customerFact?.status === 'ready' ? toStoreCustomerRows(ctx.readModels.customerFact.data) : undefined`
- `toStoreCustomerRows` は `@/application/readModels/customerFact` から import

## 3. Context Fields Consumed — 解説

- 16 field touch（WID-002 の 17 に次ぐ大規模）
- `readModels.customerFact` の `status === 'ready'` 判定を registry 行で 2 回実行（`totalCustomers` + `storeCustomerMap` 導出）
- `currentCtsQuantity?.byDay` / `?.byStore` の 2 sub-path optional 転送

## 4. Composition

- 直接の子: `PerformanceIndexChart`（`@/presentation/components/charts`）
- 間接依存: `toStoreCustomerRows`（readModel helper）

## 5. Visibility

- `isVisible` 未指定（常時可視）

## 6. Invariants

- PI 値 / 偏差値 / Z スコアの計算不変は子 chart + domain 計算に従属（`calculateAmountPI` / `calculateQuantityPI` 正本）
- `readModels.customerFact.status` が 'ready' なら `data.grandTotalCustomers` は `result.totalCustomers` の正本上書き先

## 7. Co-Change Impact

- 16 field のいずれかの型変更
- `readModels.customerFact` の status / data 構造変更
- `toStoreCustomerRows` の signature 変更
- `PerformanceIndexChart` の props shape 変更

## 8. Guard / Rule References

- `responsibilityTagGuard`（R:component）
- `presentationIsolationGuard`
- `piValuePathGuard`（PI 値の取得経路強制）
- `customerFactPathGuard`（`customerFact` の経路強制）

## 9. Pipeline Concerns / Upstream Requests

- **registry 行内 IIFE の多用**: readModel ready 判定 + 値選択を registry 行で 2 回実行。「registry は登録だけ」の期待を逸脱している。該当責務層: application（`useReadModelSliceSelector` 等で統合 selector 化の可否）
- **`readModels.customerFact.status === 'ready'` 判定の widget 側実装**: discriminated union の ready 判定を widget 側で繰り返す pattern（他 widget でも同様）。該当責務層: application（ready 判定 helper の共通化）
- **`totalCustomers` の fallback 選択**: `readModels` が ready なら `grandTotalCustomers`、そうでなければ `result.totalCustomers` という優先順序が registry 行に埋没。該当責務層: application（fallback logic の readModel hook 化）
- **`toStoreCustomerRows` が application 層 import**: registry 行が application 層関数を直接 import し call。presentation 層から application 関数への依存が surface。該当責務層: application（`storeCustomerMap` を readModel data に直接含める可否）
- **`currentCtsQuantity?.byDay` / `?.byStore` の deep optional chaining**: widget 側で deep nullable を扱う。該当責務層: application（`CurrentCtsQuantity` 型の必須化または null object 化）
