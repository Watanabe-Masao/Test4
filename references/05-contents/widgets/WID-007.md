# WID-007 — 天気-売上 相関分析

## 1. 概要

`WeatherWidget` に full ctx を passthrough し、天気と売上の相関を描画する registry entry（registryChartWidgets.tsx:133-137）。

## 2. Data Acquisition Path

- registry 行では `ctx` を full passthrough
- 実際のデータ取得・描画は全て `WeatherWidget` 内部に委譲

## 3. Context Fields Consumed — 解説

- registry 行からの直接参照 0（full ctx passthrough）
- `WeatherWidget` は内部で `ctx.weatherDaily` / `prevYearWeatherDaily` / `weatherPersist` を触ると想定されるが、registry 層では不可視

## 4. Composition

- 直接の子: `WeatherWidget`（`./WeatherWidget` から import、:10）

## 5. Visibility

- `isVisible` 未指定（常時可視）
- 天気データ不在時は子 `WeatherWidget` 内部で handle

## 6. Invariants

- 本 widget 自身は数学的不変条件を持たない
- 天気データの契約は `weatherCorrelationProjectionGuard` で管理（CLAUDE.md 2026-04-15 変更）

## 7. Co-Change Impact

- `WidgetContext` の型変更（full ctx passthrough）
- `WeatherWidget` の props shape 変更（`ctx: WidgetContext` を想定）
- `weatherDaily` / `prevYearWeatherDaily` / `weatherPersist` field の型変更
- `buildDailySalesProjection` / `DailyWeatherSummary` 型変更

## 8. Guard / Rule References

- `responsibilityTagGuard`（R:component）
- `presentationIsolationGuard`
- `weatherCorrelationProjectionGuard`（天気相関の日次 projection 強制、本 widget の子が経路を使用）

## 9. Pipeline Concerns / Upstream Requests

- **full ctx passthrough の静的解析不能**: `WeatherWidget` の実 ctx 依存が registry 層で不可視。特に ETRN fallback 系の `weatherPersist` / `weatherDaily` / `prevYearWeatherDaily` が子側でどう扱われるか registry から見えない
- **`WeatherWidget` 単独 file**: `Dashboard/widgets/WeatherWidget.tsx` が単一 file に集約されており、他 chart widget（`UnifiedHeatmapWidget` 等は `UnifiedAnalyticsWidgets` に同居）と配置規約が揃っていない。該当責務層: presentation
