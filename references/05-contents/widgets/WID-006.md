# WID-006 — 売上・仕入 店舗比較

## 1. 概要

`allStoreResults.values()` を配列化し、2 店舗以上の場合のみ `SalesPurchaseComparisonChart` に comparison / stores / storeDailyLane を forward する registry entry（registryChartWidgets.tsx:111-130）。

## 2. Data Acquisition Path

- registry 行内で `Array.from(allStoreResults.values())` を実行し、`results.length < 2` で早期 null return
- `storeDailyLane?.bundle.currentSeries ?? null` で lane 参照（optional + default）
- `chartPeriodProps?.rangeStart` / `rangeEnd` を optional 転送

## 3. Context Fields Consumed — 解説

- 6 field touch（うち `storeDailyLane.bundle.currentSeries` / `chartPeriodProps.rangeStart` / `rangeEnd` は sub-path）
- registry 行で `Array.from(Map)` の配列化 + `length < 2` チェックを実行（inline logic）
- `storeDailyLane` は `ctx.storeDailyLane` optional、`.bundle.currentSeries` も optional なので二重 null-safe

## 4. Composition

- 直接の子: `SalesPurchaseComparisonChart`（`@/presentation/components/charts`）
- `storeDailyLane.bundle.currentSeries` を `storeDailySeries` prop として渡し、子は raw `StoreResult.daily` を触らず lane 経由の series を消費（`storeDailyLaneSurfaceGuard` 準拠）

## 5. Visibility

- `isVisible` 未指定（常時可視）
- ただし render 内で `results.length < 2` で **早期 null return**（widget 固有 gate）
- 実質的な可視条件は「2 店舗以上選択時のみ」

## 6. Invariants

- 2 店舗未満では render が null を返す（空 map + 1 店舗 map を同等扱い）
- 複数店舗の比較順序は `allStoreResults.values()` のイテレーション順（Map 挿入順）に従う
