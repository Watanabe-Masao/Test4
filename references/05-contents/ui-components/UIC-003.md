# UIC-003 — KPI Grid (KpiCard 並びレイアウト)

## 1. 概要

`KpiGrid` は **styled-components で生成された CSS Grid 容器**で、複数の `KpiCard`（UIC-002）を統一間隔・列数で並べるレイアウト primitive（`KpiCard.styles.ts` 由来、`KpiCard.tsx` から re-export）。

## 2. Props Contract

- `KpiGrid` は `styled.div` の薄い wrapper、props は HTML div 標準のみ（`className` / `style` / `children`）
- 列数 / gap は CSS 内で fix（responsive grid）
- 子は通常 `KpiCard` 並び（4 件 × 1 行 等）

## 3. State Ownership

- なし（pure styled wrapper）

## 4. Hook Dependencies

- なし

## 5. Side Effects

- なし

## 6. State Coverage

| state | 振る舞い |
|---|---|
| ready | 子 children を grid 配置（responsive、子が 0 件でも空 grid 表示）|
| empty/loading/error は子 KpiCard 側に閉じる（KpiGrid は容器のみ）|

## 7. Visual Evidence

- 現状: 単独 visual test なし（Storybook の KpiCard story で間接的にテスト）
- Phase G で primitive 単独の visual baseline 設置検討

## 8. Consumers

- `KpiCard` と pair で常用:
  - `CostDetailKpiSummaryWidget.tsx` (4 KpiCard 並び)
  - `DashboardPage.tsx` 系 KPI section
  - `Reports` / `Insight` ページの KPI 並び

## 9. Co-Change Impact

- `KpiCard.styles.ts` の column 数 / gap 値変更 → 全 consumer の見た目変更
- `KpiGrid` を `KpiCard.tsx` から `tables.ts` 経由 re-export している経路変更 → import 経路修正
