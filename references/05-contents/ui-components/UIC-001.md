# UIC-001 — Dashboard 状況サマリー（Enhanced）

## 1. 概要

`ConditionSummaryEnhanced` は **memo 化された React component** で、Dashboard のヘッダ部に配置される条件サマリー（経過日数 / 年月 / 比較対象 / 客数 / 当月実績 / readModels readiness 等）を 1 行で表示する。WID-001 が `ctx={ctx}` 経由で props を流し込み、SP-A ADR-A-001 で `Pick<DashboardWidgetContext, ...>` 型に narrow 化された **公開契約コンポーネント**。

## 2. Props Contract

```ts
type ConditionSummaryEnhancedProps = Pick<
  DashboardWidgetContext,
  // 約 8 field の Pick<> (詳細 source line 73)
  'elapsedDays' | 'year' | 'month' | 'comparisonScope' |
  'currentCtsQuantity' | 'result' | 'readModels' | 'queryExecutor'
>
```

- 全 field が `DashboardWidgetContext` 由来 → ctx 経由で flow
- ADR-A-001 PR2-PR3 で `widgetContext` rename + `Pick<>` 化、registry layer から個別 props に展開しないことを `fullCtxPassthroughGuard` で強制 (baseline=0 fixed mode)

## 3. State Ownership

- 内部 state なし（pure presentation、memo 化のみ）
- `ConditionSummaryEnhancedRows` 子に props を forward して描画

## 4. Hook Dependencies

- 直接の hook 使用なし（memo + 子 component の組み合わせ）
- `readModels.customerFact` の status は `discriminated union` で渡され、子で readiness 判定

## 5. Side Effects

- なし（pure presentation）

## 6. State Coverage

| state | 振る舞い |
|---|---|
| empty | `result` 空でも default 表示（年月 + 経過日数のみ）|
| ready | 全 props そろい完全表示 |
| loading | `readModels.customerFact.status === 'pending'` で legacy `result.totalCustomers` fallback |
| error | `readModels.customerFact.status === 'error'` で fallback 表示継続 |
