設計ルール

  • 新規チャート実装時は 必ず theme.chart.semantic.* から色を拾う。hex をインラインで書かない
  • 概念名で書く (「売上」は semantic.sales)。palette 直接参照 (palette.primary) はしない
  • 前年比較は当年色 + 60% alpha の *Prev を使用。破線で描画
  • trend (↑↓) には sc.positive / sc.negative (CUD)、棒の正負には chart.semantic.positive / negative (status) という住み分け
  • 新しい業務概念が必要になったら ChartSemanticColors interface に追加 → 再ビルドで型チェックが通ることを確認
